# 代码文件说明（folder.md）

本文件声明本目录下**全部代码文件**的内容、功能与输入输出。

- `run-agent.js`
  - 功能：启动自主探索代理主循环
  - 输入：`.env` 环境变量
  - 输出：持续运行的代理进程，更新 `state/` 与 `journal/`
- `run-monitor.js`
  - 功能：启动监控 CLI
  - 输入：`.env` 环境变量与 `state/` 输出文件
  - 输出：终端实时仪表盘
- `package.json`
  - 功能：声明 Node.js 依赖与 npm 脚本
  - 输入：无（供 npm 读取）
  - 输出：可执行的 `npm run` 命令与依赖清单
- `start.sh`
  - 功能：一键启动脚本（创建环境、安装依赖并启动）
  - 输入：当前目录与可选 `.env`
  - 输出：依赖安装结果与启动进程
- `src/config.js`
  - 功能：加载与解析配置
  - 输入：环境变量与 `.env`
  - 输出：配置对象
- `src/prompts.js`
  - 功能：存放系统提示词与用户提示模板
  - 输入：日志上下文、命令输出（在模板渲染时填充）
  - 输出：完整提示词字符串
- `src/llmClient.js`
  - 功能：调用 vLLM OpenAI 兼容 API，并带重试
  - 输入：提示词与生成参数
  - 输出：模型文本响应
- `src/journal.js`
  - 功能：写入/读取日志 Markdown
  - 输入：日志内容、字符长度限制
  - 输出：`journal/` 中的日志文件与上下文摘要
- `src/agent.js`
  - 功能：主循环控制、命令执行、状态写入
  - 输入：配置、日志上下文、模型响应
  - 输出：日志与 `state/` 状态文件
- `src/monitor.js`
  - 功能：读取状态与日志并渲染终端 UI
  - 输入：`state/` 文件与日志片段
  - 输出：实时监控界面
- `src/utils.js`
  - 功能：通用解析与文件工具
  - 输入：字符串/路径
  - 输出：解析后的值或截断文本
